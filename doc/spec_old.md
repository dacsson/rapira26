## 1. Обзор на примерах

Рапира — язык программирования [1, с. 199–206], разработанный Г.А.Звенигородским как средство перехода от учебного языка программирования Робик [1] к стандартным языкам программирования. Один из интерпретаторов для этой версии языка был реализован на ПЭВМ Агат в системе "Школьница" [2]. Опыт использования системы "Школьница" позволил начать разработку новой версии языка. Система программирования для новой версии Рапиры реализуется на ПЭВМ Yamaha и Корвет.

В настоящей статье приведен обзор новой версии языка. Язык показан на примерах протоколов работы небольших программ, иллюстрирующих в основном работу с объектами (вывод программы отмечен символом `◆`). Предписания языка Рапира типичны для языков процедурного типа, поэтому в примерах они используются почти без комментариев. (Комментарии в Рапире записываются от символа `\` до конца строки. В настоящей статье комментарии, относящиеся к программе, а не к конструкциям языка, дополнительно выделены символом `▪`).

### 1.1. Работа в системе Рапира

Программ на Рапире — при работе в системе — фактически нет. Есть описания процедур и функций и есть отдельные предписания, исполняемые в диалоге. Исполнение предписаний идет в рабочей области памяти. Рабочая область создается при входе в систему и может быть запомнена при выходе из нее. В рабочую область включаются, в частности, все глобальные переменные со значениями-объектами.

### 1.2. Объекты и операции над ними

Объекты: логические (да, нет), целые и вещественные числа, тексты, кортежи, пустой объект (пусто). Объектами также считаются процедуры и функции, модули и устройства. Для всех объектов определены операции `=` и `/=`.

Логические объекты: да, нет. Операции над ними: `и`, `или`, `не`.

Числа:
```
125                \ целое
530.84             \ вещественное
1.9е-8             \ вещественное
```

Операции над числами: `+` и `-` (унарные и бинарные), `*`, `/`, `**` (возведение в степень), `//` и `/%` (деление нацело и остаток от деления нацело — для целых чисел), `>`, `<`, `>=`, `<=`.

#### Пример 1.

```
функ ПРОСТОЕ(N)             \▪ N — простое число или нет?
  если N<2 то
    возврат нет             \ возврат значения из функции
  все
  для M от 2 до корень(N)+0.5 цикл
    если N /% M = 0 то
      возврат нет
    все
  кц
  возврат да
конец
вывод: ПРОСТОЕ(2003)        \ вывод на экран
◆ да
```

Тексты:
```
"В одном сказочном городе жили коротышки"
""                         \ пустой текст
" 2 * 2 = 11 "
```

Операции над текстами: `#` (длина), `+` (склеивание), `*` (умножение на целое неотрицательное число), `[]` (выборка символа), `[:]` (вырезка подтекста).

#### Пример 2.

```

  вывод: "@" * N               \   @@@@@
  повтор (N-2) цикл
    вывод: "@" + " " * (N-2) + "@"  \   @   @
  кц
  вывод: "@" * N               \   @@@@@
конец
вызов РАМКА(5)
◆ @@@@@
◆ @   @
◆ @   @
◆ @   @
◆ @@@@@
```

#### Пример 3.

```
проц ШИВОРОТ_НАВЫВОРОТ ()      \▪ печать текста наоборот
  вывод: "Введите текст"       \▪ приглашение ко вводу
  ввод текста: ТЕКСТ            \ запрос текста в диалоге
  НАОБОРОТ:= ""                \ пустой текст
  для К до #ТЕКСТ цикл         \ от 1

```

#### Пример 4.

```
проц ТЕКСТ_ПО_СЛОВАМ (ТЕКСТ)      \▪ печать текста по словам
  ТЕКСТ:= ТЕКСТ + " "             \▪ — это для обработки
                                   \ последнего слова
  пока ТЕКСТ /= "" цикл
    К:= индекс(" ", ТЕКСТ)        \ номер первого вхождения
                                   \ " " в ТЕКСТ (или 0)

      вывод: ТЕКСТ[:К-1]          \ "аб вг д"[:2] = "аб"

Кортежи:
```
◄ 2, 3, 5, 7, 11 ►
◄ "а", "е", "и", "о" ►
◄ ◄ 1, 4, ◄ 6, 7, 9 ► ►, 10, ◄ 44, 70, 99 ► ►
```

Кортеж — упорядоченная последовательность элементов. Элементом кортежа может быть любой объект языка (в том числе и кортеж). Кортеж строится из объектов с помощью операции формирования `◄ ►`. Операции над кортежами: `#` (длина), `+` (склеивание), `*` (умножение на целое неотрицательное число), `[]` (выборка элемента кортежа), `[:]` (вырезка подкортежа).

```

КОРТЕЖ_ИЗ_СЛОВ:= ТЕКСТ_НА_СЛОВА

```

### 1.3. Переменные

Значением переменной может быть любой объект языка.

#### Пример 8.

Игра: машина пытается с помощью вопросов угадать задуманного человеком зверя. (Человек отвечает на вопросы "да" или "нет"). Если зверь машине неизвестен, то она с помощью человека пополняет свои знания.

Алгоритм. Знания о животных представляются в виде дерева:

```
        вопрос
        /    \
   да  /      \ нет
      /        \
   вопрос     ...
    /   \
   /     \
да/    нет\     зверь
  ...   .../
       /
зверь ...... зверь
```

Человеку задается вопрос. В зависимости от ответа выбирается правое или левое поддерево. Так до тех пор, пока не будет выбрано название зверя.

Если это не тот зверь, то этот лист заменяется на поддерево:
```
  вопрос            или          вопрос
  /    \                         /    \
зверь  зверь-               зверь    зверь
       -загадка                      -загадка
```

Загаданный зверь и вопрос, по ответу на который он распознается, запрашиваются у человека.

Реализация. Дерево представляется кортежем ЗНАНИЯ:

```
◄ вопрос, поддерево для 'да', поддерево для 'нет' ►
```

Каждое поддерево — это такой же кортеж или текст — название зверя. При выборе поддеревьев значение переменной ПОПЫТКА — кортеж или текст, в зависимости от глубины спуска по дереву.

```
проц ЗВЕРИ ( <=ЗНАНИЯ )      \▪ игра: машина угадывает
                               \▪ задуманного зверя
  вывод: ЗНАНИЯ[1], "(да,нет)" \▪ машина задает вопрос
  ввод: ОТВЕТ                  \ ввод объекта в диалоге
  если ОТВЕТ то
    Н:= 2
  иначе
    Н:= 3
  все
  ПОПЫТКА:= ЗНАНИЯ[Н]         \▪ сокращение вариантов
  если тип_корт(ПОПЫТКА) то   \▪ еще есть выбор
    ЗВЕРИ(<=ПОПЫТКА)
  иначе                       \▪ остался один зверь
    вывод: "Это ", ПОПЫТКА, "?(да,нет)"
    ввод: ОТВЕТ
    если ОТВЕТ то              \▪ зверь угадан
      вывод: "Ура!"
    иначе                      \▪ этот зверь неизвестен
                               \▪ надо пополнить знания
      вывод: "Сдаюсь. Что это за зверь?"
      ввод текста: ЗВЕРЬ
      вывод: "Введите вопрос, с помощью"
      вывод: "которого можно отличить"
      вывод: ЗВЕРЬ, " и ", ПОПЫТКА, ":"
      ввод текста: ВОПРОС
      вывод: "Ответ для ", ЗВЕРЬ, "?"
      ввод: ОТВЕТ
      если ОТВЕТ то
        ПОПЫТКА:= ◄ ВОПРОС, ЗВЕРЬ, ПОПЫТКА ►
      иначе
        ПОПЫТКА:= ◄ ВОПРОС, ПОПЫТКА, ЗВЕРЬ ►
      все
    все
  все
  ЗНАНИЯ[Н]:= ПОПЫТКА
конец
ВСЕ_ЗВЕРИ:= ◄ "Живет в воде?", "рыба", "страус" ►
цикл
  ЗВЕРИ(<=ВСЕ_ЗВЕРИ)
кц
```

(Протокол работы: см. оригинал, с. 10–11)

### 1.4. Предписания

Предписания: присваивание, вызов процедуры, условное предписание, выборы, циклы, вывод, ввод, выход из цикла, возврат из процедуры или из функции. Почти все предписания уже встречались в приведенных примерах.

### 1.5. Процедуры и функции

В процедуре (функции) все нелокальные имена должны быть перечислены в списке "чужие". (Локальные имена можно перечислить в списке "свои", но это необязательно). Имена, используемые в данной процедуре, известны и в вызываемых из нее процедурах (если там они указаны как "чужие"), так как поиск "чужих" имен ведется по динамической цепочке вызовов (см. Пример 10).

Параметры процедур:
- входные: `=>ЧТО`, `ТЕКСТ` (стрелка `=>` необязательна) — см. Примеры 9, 4
- возвратные: `<=ГДЕ`, `<=ЗНАНИЯ` — см. Примеры 9, 8

У функций параметры только входные.

### 1.6. Модули и устройства

Планируемые возможности аппарата модулей и устройств:
- подключение модуля пополняет список стандартных имен (в том числе имен стандартных процедур и функций);
- подключение модуля пополняет набор простых предписаний (т.е. языковыми средствами создаются исполнители, аналогичные существующим в Робике [1]);
- устройство есть частный случай модуля. С устройством возможен обмен через предписания ввода/вывода:
  ```
  вывод >>принтер: "Этот текст — на принтер"
  ```

В предписаниях ввода/вывода могут быть указаны различные режимы ввода/вывода:
```
ввод текста: ВОПРОС             \ см. Пример 8
```
"текста" — ключевое слово для устройства КЛАВ (клавиатура).

## 2. Рапира. Формальное описание языка (без модулей, устройств и обработки исключительных ситуаций)

### 2.1. Синтаксис

**Алфавит:**
- буквы: большие и малые, русские и латинские;
- цифры;
- спецзнаки:
  ```
  .  ,  :  ;  (  )  [  ]  <  >  =
  +  -  *  /  %  #  _  "  \  ◄  ►
  ```
- и все, что угодно.

```
разделитель ::= ( пробел | перевод_строки | комментарий )
               комментарий ::= '\' { литера* } перевод_строки
лексема ::= спецсимвол | ключевое_слово | имя |
             целое_без_знака | вещественное_без_знака | текст

спецсимвол ::= ;  ,  :  (  )  [  ]  ◄  ►
               :=  =>  <  >  <=  >=  =  /=  #
               +   -   *  /  //  /%  **
```
\* кроме перевода строки

```
ключевое_слово ::= бпс    вызов   или   по     текста
                   ввод   выход   иначе повтор  то
                   возврат для    конец  пока    функ
                   все    до      кц     при     цикл
                   выбор  если    не     проц    чужие
                   вывод  и       от     свои    шаг

имя ::= буква { буква | '_' | цифра }

целое ::= [ '+' | '-' ] целое_без_знака
целое_без_знака ::= цифра { цифра }

вещественное ::= [ '+' | '-' ] вещественное_без_знака
вещественное_без_знака ::= цифра { цифра }
                           ( '.' цифра { цифра } ) [ 'e' целое |
                                                      'е' целое ]

текст ::= '"' { литера* | '""' } '"'
```
\* кроме `"` и перевода строки

```
условие ::= выражение
выражение ::=
    выр_а { ( 'и' | 'или' ) выр_а }

выр_а ::=
    [ 'не' ]
    выр_б
    { ( '>' | '<' | '>=' | '<=' | '=' | '/=' ) выр_б }

выр_б ::=
    [ '+' | '-' ]
    выр_в
    { ( '**' | '*' | '/' | '//' | '/%' | '+' | '-' ) выр_в }

выр_в ::=
    [ '#' ]
    ( имя |
      текст |
      целое_без_знака |
      вещественное_без_знака |
      процедура |
      функция |
      ( форм_корт ::= '◄' [ л_выр ] '►' ) |
      '(' выражение ')' )
    { ( выб_выр ::= '[' л_выр ']' |
                    '[' [ выражение ] ':'
                        [ выражение ] ']' ) |
      факт_парам_функ }

л_выр ::= выражение { ',' выражение }

факт_парам_функ ::=
    '(' [ [ '=>' ] выражение
         { ',' [ '=>' ] выражение } ] ')'

процедура ::=
    'проц' [ имя ]
    '(' [ [ '=>' | '<=' ] имя
         { ',' [ '=>' | '<=' ] имя } ] ')' ;;
    [ описание_имен ]
    предп
    'конец'

;; ::= ( ';' | перевод_строки )

описание_имен ::= (  'чужие' ':' имя { ',' имя } ;; |
                   [  'свои'  ':' имя { ',' имя } ;; ] ) |
                   (  'свои'  ':' имя { ',' имя } ;; ) |
                   [ 'чужие'  ':' имя { ',' имя } ;; ]

функция ::=
    'функ' [ имя ]
    '(' [ [ '=>' ] имя
         { ',' [ '=>' ] имя } ] ')' ;;
    [ описание_имен ]
    предп
    'конец'

единица_общения_с_системой ::=
    ( предписание | описание ) ;;

предп ::= { предписание ;; }
описание ::= ( процедура | функция )

предписание ::=
    ( пустое ::= '' ) |
    присваивание |
    вызов_процедуры |
    условное |
    выбор |
    цикл |
    вывод |
    ввод |
    ( выход_из_цикла ::= 'выход' ) |
    ( выход_из_процедуры ::= 'возврат' ) |
    ( выход_из_функции ::= 'возврат' выражение )

присваивание ::=
    ( переменная ::= имя { выб_выр } ) ':=' выражение

вызов_процедуры ::=
    ( 'вызов' выражение | имя ) факт_парам_проц

факт_парам_проц ::=
    '(' [ ( '<=' переменная |
            [ '=>' ] выражение )
          { ',' ( '<=' переменная |
                  [ '=>' ] выражение ) } ] ')'

условное ::=
    'если' условие
        'то' предп
        [ 'иначе' предп ]
    'все'

выбор ::=
    ( 'выбор' выражение
        ( 'при' выражение { ',' выражение } ':' предп )
    |
      'выбор'
        ( 'при' условие ':' предп )
    )
    [ 'иначе' предп ]
    'все'

цикл ::=
    [ ( 'для' имя [ 'от' выражение ]
                   [ 'до' выражение ]
                   [ 'шаг' выражение ] ) |
      'повтор' выражение              ]
    [ 'пока' условие                  ]
    'цикл'
        предп
    'кц' [ 'по' условие ]

вывод ::=
    'вывод' [ 'бпс' ] [ ':' выражение { ',' выражение } ]

ввод ::=
    'ввод' [ 'текста' ] ':' переменная { ',' переменная }
```

**Контекстные условия:**
- предписание "выход_из_цикла" может стоять только в цикле;
- предписание "выход_из_процедуры" может стоять только в процедуре;
- предписание "выход_из_функции" может стоять только в функции;
- в процедуре и функции: все имена в списках "чужие", "свои", в списке параметров и имя процедуры (функции) — все различны;
- в "описании" (см. "единица_общения_с_системой") имя процедуры (функции) обязательно.

### 2.2. Семантика

#### 2.2.1. Имена и переменные

Значением переменной может быть любой объект языка. Существуют стандартные имена (см. п. 2.3). Правила локализации имен см. п. 2.2.2 (процедуры и функции). Если имя в данной области локализации не является стандартным и ему не было присвоено значение, то его значением является пустой объект (пусто).

#### 2.2.2. Объекты

| Объект | Обозначение |
|--------|------------|
| пустой | `'пусто'` |
| логический | `'да'` \| `'нет'` |
| процедура | процедура (см. синтаксис) |
| функция | функция (см. синтаксис) |
| целый | целое \| целое_без_знака (см. синтаксис) |
| вещественный | вещественное \| вещественное_без_знака (см. синтаксис) |
| текст | текст (см. синтаксис) |
| кортеж | `'◄'` [ обозначение_объекта { `','` обозначение_объекта } ] `'►'` |

Текст. Последовательность `""` в обозначении текста представляет литеру `"` в тексте.

Кортеж — упорядоченный набор произвольных объектов.

Процедуры и функции. Имя процедуры (функции) локально в процедуре (функции).

Параметры:
- входные — `[ '=>' ] имя`
- возвратные — `'<=' имя`

У функций параметры только входные. Описания имен: по умолчанию имя, не являющееся стандартным, считается "своим" (локальным). Правило локализации имен: по динамической цепочке вызовов.

```

**
*  /  //  /%
-  +  ( унарные и бинарные )
>  <  >=  <=

не
и
или
```

#### 2.2.4. Операции, предписания и описания

Семантика большинства конструкций языка описана в терминах самого языка с использованием базисного набора предписаний и операций. Семантика базисного набора описана словесно.

##### 2.2.4.1. Обозначения

```
a, b, c, z, bj, f, fj, g, k, h, p, n, x  — выражения
i                                           — имя
v                                           — переменная
S, Sj                                       — предписания
```

##### 2.2.4.2. Базисный набор

**Операции:**

`p( ... )` — `p` функция — вызов функции.

`◄ ... ►` — формирование кортежа.

`#k` — `k` текст, кортеж — длина.

`k [ a ]` — `k` текст, кортеж; `a` целое, `1 <= a <= #k` — выборка.
Результат: элемент кортежа с номером `a` (символ текста с номером `a`).

`a ** b` — `a, b` — числа — возведение в степень.
- `b` — вещественное: `a ** b = exp(ln(a) * b)`
- `b` — целое: см. частичное сведение к базису.

`a * b` — `a, b` — числа; `a` текст/кортеж, `b` целое; `b` текст/кортеж, `a` целое — умножение. Умножение на целое число (см. частичное сведение к базису).

`a / b` — `a, b` — числа — деление.
Результат: `a, b` — целые: целый или вещественный; в остальных случаях — вещественный.

`-a` — `a` число — противоположное значение.

`a + b` — `a, b` числа; `a, b` тексты; `a, b` кортежи — сложение чисел, склеивание текстов и кортежей.

`a > b` — `a, b` числа — сравнение "больше".

`a = b` — `a, b` — любые — сравнение на равенство; случай `a, b` — кортежи: см. частичное сведение к базису.

**Предписания:**

`i := x` — `x` любой объект — присваивание значения к имени `i`.

`вызов p( ... )` — `p` процедура — вызов процедуры.

`если f то S все` — `f` логическое — условное предписание.

`вывод: ...` — вывод объектов. Выводятся (на экран) обозначения объектов (кроме текстов, которые выводятся без кавычек).

`вывод бпс: ...` — вывод без перевода строки (здесь временно, потом в устройстве).

`ввод: ...` — ввод объектов. Вводятся (с клавиатуры) обозначения объектов.

`ввод текста: ...` — ввод текста без кавычек (здесь временно, потом в устройстве).

`возврат` — возврат из процедуры.

`возврат x` — `x` любой объект — возврат значения из функции.

##### 2.2.4.3. Конструкции, сводимые к базису

Определяющие конструкции построены так, что для неверных аргументов дают ошибку, причем не позже, чем необходимо. Все новые имена должны отличаться от имен в смысле Рапиры (в приведенных фрагментах это не так для удобства чтения).

Дополнительно использованы:
- процедура ОШИБКА (она могла бы быть и с параметром — какая ошибка),
- стандартные функции ТИП_ЦЕЛ, ...
- мифические предписания "ЗАЩИТИТЬ имя" (т.е. запретить присваивания этому имени) и "СНЯТЬ ЗАЩИТУ имя", без которых не описывается цикл ДЛЯ.

**1. Равенство для кортежей:**
```
k = h  ≡
    функ РАВ_КОРТ (К, Н)      \ рекурсивное место
      если #К /= #Н то
        возврат нет
      все
      для ИНД до #К цикл
        если К[ИНД] /= Н[ИНД] то
          возврат нет
        все
      кц
      возврат да
    конец
            (k, h)
```

**2. Умножение на целое число** (текста или кортежа — только на неотрицательное число):
```
a * h  ≡
    функ ЦЕЛ_УМН (АА, ВВ)
      если тип_цел(АА) то
        А:= АА ; В:= ВВ
      иначе
        А:= ВВ ; В:= АА
      все
      выбор
        при тип_цел(В)  : ПР:= 0
        при тип_вещ(В)   : ПР:= 0.0
        при тип_текст(В) : ПР:= ""
        при тип_корт(В)  : ПР:= ◄ ►
      все
      если А>=0 то  У:= А
      иначе         У:= -А
      все
      повтор У цикл
        ПР:= ПР+В
      кц
      если А>=0 то возврат ПР
      иначе        возврат -ПР  \ для текста или
      все                       \ кортежа — ошибка
    конец
            (a, b)
```

**3. Возведение в целую степень:**
```
a ** b  ≡
    функ ЦЕЛ_СТЕП (А, В)
      если А=0 и В=0 то
        ошибка()
      все
      если В>0 то  N:= В
      иначе        N:= -В
      все
      Р:= 1
      пока N>0 цикл
        если N/%2 /= 0 то
          Р:= Р*А
        все
        А:= А*А
        N:= N//2
      кц
      если В>0 то возврат Р
      иначе       возврат 1/Р
      все
    конец
            (a, b)
```

**Сведение конструкций к базисным.**

**Предписания:**

**1) вызов процедуры:**
```
i ( ... )  ≡  вызов i ( ... )             \ i — процедура
```

**2) условное:**
```
если f то S1
  иначе     S2
все             ≡
    вызов
      проц ЕСЛИ_ИНАЧЕ (F)
        чужие : ...     \ все имена из S1, S2 и f
        если F то
          S1
          возврат
        все
        S2
      конец
            (f)         \ f — логическое
```

**3) выбор:**

Форма с условиями:
```
выбор
  при f1 : S1
  при f2 : S2
  ...
  при fn : Sn
  иначе    S0
все         ≡
    если      f1 то S1
    иначе если f2 то S2
    иначе ...
              если fn то Sn
              иначе       S0
    все все ... все
```

Форма с выражением:
```
выбор a
  при b1 : S1
  при b2 : S2
  ...
  при bn : Sn
  иначе    S0
все         ≡
    вызов
      проц ВЫБОР_ВЫР (А)
        чужие : ...     \ все имена из S1, bi и a
        если      A = b1 то S1
        иначе если A = b2 то S2
        иначе ...
                  если A = bn то Sn
                  иначе           S0
        все все ... все
      конец
            (a)         \ a — любое
```

Множественные значения при:
```
выбор a
  при b1, b2 : S1
  ...
все         ≡
    выбор a
      при b1: S1
      при b2: S1
      ...
    все
```

**Операции:**

**4) не f:**
```
≡  функ ЛОГ_НЕ (F)
     если F то возврат да
     иначе     возврат нет
   все
   конец
        (f)
```

**5) f и g:**
```
≡  функ ЛОГ_И ( )
     чужие: ...        \ все имена из f, g
     если не f то
       возврат нет
     иначе
       если g то возврат да
       иначе     возврат нет
       все
     все
   конец
        ( )            \ f, g логические
```

**6) f или g:**
```
≡  функ ЛОГ_ИЛИ ( )
     чужие: ...        \ все имена из f, g
     если f то
       возврат да
     иначе
       если g то возврат да
       иначе     возврат нет
       все
     все
   конец
        ( )            \ f, g логические
```

**7) +a:**
```
≡  0+a
```

**8) a - b:**
```
≡  a + (-b)
```

**9) a < b:**
```
≡  функ МЕНЬШЕ (А, В)
     возврат не(А>В или А=В)
   конец
        (a, b)
```

**10) a >= b:**
```
≡  не(a<b)
```

**11) a <= b:**
```
≡  не(a>b)
```

**12) a /= b:**
```
≡  не(a=b)
```

**13) цикл, выход из цикла:**
```
цикл
  S
кц          ≡
    вызов
      проц БЕСКОНЕЧНО ( )
        чужие: ...      \ все имена из S
        S
        БЕСКОНЕЧНО()
      конец
        ( )
```

```
выход       ≡  возврат              \ если цикл — как выше
```

```
пока f цикл
  S
кц по g     ≡
    цикл
      если не(f) то выход все
      S
      если g то выход все
    кц
```

```
пока f цикл
  S
кц          ≡
    пока f цикл
      S
    кц по нет
```

```
цикл
  S
кц по f     ≡
    пока да цикл
      S
    кц по f
```

```

  пока f цикл
  S

    вызов
      проц ЦИКЛ_ПОВТОР (N)
        чужие: ...      \ все имена из S, f и g
        если не(тип_цел(N) и N>=0) то
          ошибка()
        все
        пока N>0 и f цикл
          S
          N:= N-1
        кц по g
      конец
            (n)         \ n целое
```

```
повтор n цикл
  S
кц          ≡
    повтор n
      пока да цикл
      S
    кц по нет
```

```

        если не(тип_цел(А) или тип_вещ(А)) то

        все
        если не(тип_цел(С) или тип_вещ(С)) то

        все

          i:= i+С
          защитить i
        кц по g
        снять защиту i
      конец
            (a, c)      \ a, c — числа
```

```

кц          ≡
    для i от a шаг с
      пока да цикл
      S
    кц по нет
```

    для i от 1 шаг с цикл

```
для i от a до b шаг с
  пока f цикл
  S

    вызов

## 3. Рапира. Отличия в реализации на Корвете и Yamaha по сравнению с описанием языка

### 3.1. Синтаксис

Алфавит. Не различаются следующие русские и латинские буквы:
```
К Е Н Х В А Р О С М Т
е     х     а       с
```

Вне констант-текстов не различаются большие и малые буквы.

Лексемы. Ключевые слова зарезервированы.

```
имя  ::= ( буква | '_' ) { буква | '_' | цифра }
текст ::= '"' { '^' буква |
                '\' цифра [ цифра ] ] |
                литера* |
                '""' } '"'
```
\* кроме `"` и перевода строки

Кроме спецсимволов `◄` и `►` есть эквивалентные им `<*` и `*>`.

Процедуры и функции. Нет обозначения_объекта-процедуры и обозначения_объекта-функции, есть только описание процедур и функций на уровне "единицы_общения_с_системой".

**Предписания.**
```
вывод ::= 'вывод' { 'бпс' }
          [ ':' выражение { ',' выражение } ]
ввод  ::= 'ввод' { 'текста' | 'пригл' выражение } ':'
          переменная { ',' переменная }
```

Часть предписания "вывод" ("ввод") до `:` не должна разбиваться переводами строк.

### 3.2. Семантика

Объекты. В тексте:
- `^` буква — обозначает контрольный символ;
- `\` число — обозначает литеру с данным кодом.

Предписания. Ввод — не только обозначений объектов, а любых выражений, через запятые. "Ввод текста" — ввод только одного текста. "Ввод пригл" — ввод с приглашением.

### 3.3. Стандартные имена

Нет функций "tg", "arcsin".

Дополнительные функции:
```
код(т)    — код символа т.
алф(x)    — символ с кодом x.
клав()    — ввод одного символа (нажатием одной клавиши);
            результат — введенный символ.
нажато()  — проверка, была ли нажата клавиша;
            результат — введенный символ или пустой текст.
```

**Графика:**
- нет процедуры "обл";
- граничные координаты графического окна:
  - x-координата: 0 — 255,
  - y-координата: 0 — 191;
- цвета задаются только числами: 0–16 (0 — пустой, 16 — реверсный).
